# 멋사 백엔드 스터디 · 배포/운영 트랙 커리큘럼

> Spring 커리큘럼을 마치고 넘어온, **"내가 만든 앱을 진짜 서비스처럼 인터넷에 띄우고 운영하기"** 트랙입니다.
핵심 설계 원칙: **앞 편이 만든 불편함을 다음 편이 해결한다.** 매 편 끝에 다음 편으로 넘어갈 질문이 자연스럽게 생깁니다.
> 

---

## 전체 지도

```
[1막] 운영 기본기          [2막] 트래픽과 안정성        [3막] 진짜 인프라로
"서비스처럼 띄운다"    →    "잘 되면 생기는 문제"    →    "세트 메뉴 졸업"
Docker~백업 자동화          무중단 배포~성능             EC2~오케스트레이션
```

- **총 12~15편** 규모
- 각 막이 끝날 때마다 "여기까지만 들어도 하나는 완성"되는 구조 → 중간 합류/이탈 관리 용이
- **1막 → 2막 사이에 미니 프로젝트 회차 1개**를 넣어 난이도 점프를 완충

---

## 1막 — 운영 기본기 완성

> 테마: **내 앱 하나를 진짜 서비스처럼 띄우고 운영한다.**
각 편이 비교적 독립적이라 순서 조정·회차 조절이 쉬운 구간.
> 

| 회차 | 제목 | 해결하는 불편함 | 핵심 도구 |
| --- | --- | --- | --- |
|  | Docker 편 | "내 노트북에서만 돌아요" | Docker, docker compose |
|  | Lightsail 배포 편 | "어디에 24시간 올리지?" | AWS Lightsail |
|  | 도메인 & HTTPS 편 | "IP 못생겼고 자물쇠 없음" | Caddy, Let's Encrypt |
|  | **CI/CD 편** | "고칠 때마다 손으로 배포 귀찮음" | GitHub Actions |
|  | **모니터링 & 알림 편** | "죽으면 어떻게 알지?" | 헬스체크, 디스코드/슬랙 웹훅 |
|  | **로그 관리 편** | "logs를 언제까지 손으로 보냐" | 로그 수집·검색 |
|  | **DB 백업 자동화 편** | "데이터 날아가면 끝인데?" | cron, mysqldump |
|  | **미니 프로젝트 회차** | "혼자 처음부터 해보기" | (1막 전체 복습) |

> 💡 미니 프로젝트 회차: 지금까지 배운 배포 흐름(Docker → Lightsail → 도메인 → CI/CD)을 **새 앱으로 처음부터 혼자** 해보기. 손에 익히는 시간을 주고, 2막 난이도 점프 전에 자연스러운 텀을 만든다.
> 

**1막 완료 시점:** "내 앱 하나가 자동 배포되고, 죽으면 알림 오고, 데이터도 백업된다" = 1인 서비스 운영 완성.

---

## 2막 — 트래픽과 안정성

> 테마: **서비스가 잘 되기 시작하면 생기는 문제들.**
여기서부터 Spring을 **운영 관점에서 다시** 만난다
> 

| 회차 | 제목 | 해결하는 불편함 | 핵심 개념 |
| --- | --- | --- | --- |
|  | **무중단 배포 편** | "재배포할 때 잠깐 끊김" | blue-green, `/actuator/health` |
|  | **리버스 프록시 심화 편** | "앱 하나로 트래픽 감당 안 됨" | 로드밸런싱, 앱 다중화 |
|  | **환경 분리 & 시크릿 관리 편** | ".env 하나로 버티기 한계" | dev/prod 분리, 시크릿 관리 |
|  | **캐싱 & 성능 기초 편** | "같은 요청에 매번 DB 때림" | Redis 캐싱(운영 관점) |

> 무중단 배포에서 앱 쪽 헬스체크 엔드포인트(`/actuator/health`)가 처음 등장. 이게 "이미 아는 Spring을 인프라와 연결하는" 첫 지점. 리버스 프록시(Caddy)가 이미 깔려 있어서 blue-green 흐름으로 자연스럽게 이어짐.
> 

> **1막 → 2막 난이도 점프 주의.** 무중단 배포부터 확 어려워진다. 앞의 미니 프로젝트 회차로 완충하고, 필요하면 2막 첫 편에서 개념 워밍업 시간을 넉넉히 잡을 것.
> 

**2막 완료 시점:** "트래픽이 늘어도 안 끊기고, 환경이 나뉘어 있고, 성능도 챙긴다" = 실서비스 수준 운영.

---

## 3막 — 진짜 인프라로

> 테마: **Lightsail "세트 메뉴"를 졸업하고 재료를 직접 고른다.**
지난 편들이 놔둔 개념 다리(EC2 예고)를 회수하는 졸업반 구간.
> 

| 회차 | 제목 | 해결하는 불편함 | 핵심 개념 |
| --- | --- | --- | --- |
|  | **EC2로 이사 편** | "세트 메뉴로는 부족함" | EC2, VPC 기초 |
|  | **IaC 입문 편 (Terraform)** | "콘솔 클릭을 매번 반복" | 코드로 인프라 정의 |
|  | **오케스트레이션 맛보기 편** | "docker compose의 한계" | 왜 Kubernetes 얘기가 나오나 (개념만) |
|  | **🏁 회고 & 아키텍처 총정리 편** | "우리가 뭘 쌓았지?" | 전체 다이어그램, 다음 로드맵 |

**3막 완료 시점:** 전체 여정을 한 장의 다이어그램으로 정리 + "여기서 어디로 더 갈 수 있나" 로드맵 제시 = 시리즈 완결.
